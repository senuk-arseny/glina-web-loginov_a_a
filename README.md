# Глина — веб-приложение гончарной мастерской

Учебный проект летней школы: клиентское веб-приложение для записи на мастер-классы гончарной мастерской «Глина». Бэкенда нет — вся бизнес-логика (проверка мест, ранняя/поздняя отмена, лист ожидания, рейтинг мастеров, уведомления) реализована в `src/mock/api.ts` как мок-слой, повторяющий контракт `docs/ba/openapi.yaml`. Подробное объяснение архитектуры — в `docs/ba/data-model.md` и `docs/ba/api-sequence.md`.

## Запуск

Требуется Node.js 18+ (см. `.nvmrc`).

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # production-сборка в dist/
npm run lint     # eslint
npm run test     # vitest — 10 групп тестов бизнес-логики
```


## Что реализовано

- Список занятий на 7 дней вперёд с фильтром по датам
- Бронирование с прокатом инструментов, лист ожидания с видимой позицией в очереди
- Отмена брони с ранней/поздней логикой (определяется на бэкенде)
- Оценка мастера после занятия, пересчёт среднего рейтинга
- Уведомления: подтверждение записи, напоминание, форс-мажорная отмена, waitlist
- Вход по имени и телефону без SMS, лендинг-страница

Автоматические тесты — `src/mock/api.test.ts` (8 групп: бронирование, лимит программы, отменённые слоты, граница ранней/поздней отмены, прокат, лист ожидания, вход по телефону, оценка мастера) и `src/utils/format.test.ts` (2 группы: парсинг дат, форматирование времени). Плюс ручные тест-кейсы в `docs/ba/test-cases.md`.

## Структура `docs/`

```
docs/
  ba/        — требования к MVP, архитектура, схема данных, API-контракт, тест-кейсы
  features/  — документация по каждой реализованной фиче (цель → требования → реализация → промпты → проверка → commit)
  bugs/      — документация по найденным и исправленным багам (симптом → причина → фикс → проверка → commit)
```

## Структура `src/`

```
src/
  types.ts                    — общие типы данных
  mock/
    data.ts                   — сид-данные (программы, мастера, слоты)
    api.ts                    — вся бизнес-логика (условный бэкенд)
  context/SessionContext.tsx  — сессия клиента (вход/выход)
  utils/format.ts             — форматирование дат и времени
  pages/                      — экраны приложения
  components/                 — переиспользуемые UI-компоненты
```
