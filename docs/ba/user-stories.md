# user-stories.md — Гончарная мастерская «Глина»

Каждая история имеет ID (`US-XX`) и ссылку на use case, который её реализует (`Covers`). Это трассируемость: FR ссылаются на US (`functional-requirements.md`), US ссылаются на UC (`use-cases.md`), UC содержат обратную ссылку `Realizes` на US.

## US-01
**Как клиент**, я хочу видеть список занятий на ближайшую неделю, чтобы выбрать удобное время.
- Priority: Must
- AC: список показывает слоты на 7 дней вперёд по умолчанию; есть фильтр по дате; при отсутствии слотов — empty state.
- Covers: UC-1 (шаг 1)

## US-02
**Как клиент**, я хочу открыть детали занятия, чтобы понять, подходит ли оно мне.
- Priority: Must
- AC: видны программа, мастер, рейтинг, время, свободные места, доступность проката.
- Covers: UC-1 (шаг 2)

## US-03
**Как клиент**, я хочу забронировать место на занятие, чтобы не писать в директ и не ждать ответа вручную.
- Priority: Must
- AC: бронирование доступно только при наличии мест; при отсутствии — предложение листа ожидания; лимит программы не превышается; **повторная бронь того же клиента на тот же слот отклоняется** (закрывает FR-16).
- Covers: UC-1

## US-04
**Как клиент**, я хочу выбрать прокат инструментов при записи, чтобы не носить свои.
- Priority: Must
- AC: опция доступна только при наличии свободных комплектов, показана стоимость.
- Covers: UC-1

## US-05
**Как клиент**, я хочу отменить свою запись, если планы изменились.
- Priority: Must
- AC: отмена доступна из истории; система показывает раннюю/позднюю отмену; факт поздней отмены фиксируется.
- Covers: UC-2

## US-06
**Как клиент**, я хочу встать в лист ожидания на заполненное занятие, чтобы не пропустить его.
- Priority: Must
- AC: кнопка «В лист ожидания» при отсутствии мест; уведомление при освобождении места; видна позиция в очереди.
- Covers: UC-3

## US-07
**Как клиент**, я хочу оценить мастера после занятия, чтобы поделиться впечатлением.
- Priority: Must
- AC: оценка доступна только для завершённых занятий, разовая на бронь.
- Covers: UC-4

## US-08
**Как клиент**, я хочу видеть, что занятие отменено мастерской и почему.
- Priority: Must
- AC: статус «Отменено мастерской» с причиной; повторная запись на слот недоступна (в том числе через лист ожидания).
- Covers: UC-6

## US-09
**Как клиент**, я хочу получать уведомления о записи и напоминания, чтобы не забыть прийти.
- Priority: Must
- AC: уведомление сразу после записи и напоминание перед занятием; список уведомлений доступен отдельным разделом.
- Covers: UC-7

## US-10
**Как клиент**, я хочу видеть профиль и историю посещений, чтобы отслеживать статус постоянного клиента.
- Priority: Should
- AC: история завершённых занятий, статус при 3+ посещениях, прогресс до статуса.
- Covers: UC-8

## US-11
**Как клиент**, я хочу войти в приложение по имени и телефону без лишних шагов, чтобы быстро начать пользоваться.
- Priority: Must
- AC: вход без SMS-подтверждения; повторный вход тем же телефоном возвращает тот же профиль.
- Covers: UC-5
