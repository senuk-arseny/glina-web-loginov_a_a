# Bug 01: Можно записаться на одно и то же занятие несколько раз

## Симптом
Клиент мог оформить несколько активных броней на один и тот же слот, повторно нажимая «Записаться». Найдено вручную при тестировании после первой сборки приложения.

## Требование, которое нарушалось
Не было сформулировано явно в исходном брифе — добавлено постфактум как FR-16 (`docs/ba/functional-requirements.md`), поскольку является очевидным ограничением любого бронирования (один клиент — одна активная бронь на слот).

## Причина
`createBooking()` в `src/mock/api.ts` проверял только `freeSpots`, но не проверял, есть ли у клиента уже активная бронь на этот же слот.

## Реализация фикса
- Добавлена проверка в `createBooking`:
  ```ts
  const alreadyBooked = db.bookings.some(
    (b) => b.clientId === clientId && b.slotId === slotId && b.status === "active"
  );
  if (alreadyBooked) err("already_booked", "Вы уже записаны на это занятие");
  ```
- Добавлен экспорт `hasActiveBooking(clientId, slotId)` — используется в `SlotDetailsPage.tsx`, чтобы кнопка «Записаться» вообще не показывалась, если бронь уже есть (баннер «Вы уже записаны» вместо формы)
