# Bug 02: Фильтр дат не показывает последний день диапазона

## Симптом
При выборе фильтра «с 4 по 6 июля» список занятий показывал только 4 и 5 июля — занятия 6 июля пропадали из выдачи, хотя реально существовали в расписании.

## Требование, которое нарушалось
FR-01 — «Клиент может просматривать список слотов... с фильтром по дате» (`docs/ba/functional-requirements.md`). Ожидание — включительный диапазон дат.

## Причина
`<input type="date">` возвращает строку `"2026-07-06"`. `new Date("2026-07-06")` парсит её как **UTC-полночь** этого дня. Занятие в 14:00 того же дня — позже этой полуночи, поэтому сравнение `startTime <= to.getTime()` его отсекало. Дополнительно UTC-парсинг сдвигал границу в зависимости от часового пояса браузера.

## Реализация фикса
- Добавлены `parseDateInputStart`/`parseDateInputEnd` в `src/utils/format.ts`, парсящие строку вручную (`value.split("-")`) и конструирующие **локальную** полночь/23:59:59.999 для конца дня
- `SlotListPage.tsx` использует эти функции вместо `new Date(dateFrom)`/`new Date(dateTo)` напрямую
