# Bug 03: Фонд проката инструментов не восстанавливается при отмене брони

## Симптом
Если клиент бронировал занятие с прокатом инструментов, а затем отменял бронь, комплект инструментов не возвращался в доступный фонд — `equipmentRental.availableSets` необратимо уменьшался с каждой отменённой бронью, хотя место по факту освобождалось.

## Требование, которое нарушалось
FR-04 (`docs/ba/functional-requirements.md`) косвенно подразумевает, что прокат — ограниченный, но корректно управляемый ресурс; отмена брони должна возвращать оба занятых ресурса (место и прокат) в исходное состояние.

## Причина
`cancelBooking()` в `src/mock/api.ts` восстанавливал только количество свободных мест (тогда ещё — инкрементом поля `freeSpots`), но не проверял `booking.equipmentRentalRequested` и не возвращал комплект в `equipmentRentals`.

## Реализация фикса
```ts
if (booking.equipmentRentalRequested) {
  const rental = equipmentRentals.find((r) => r.slotId === slot.id);
  if (rental) rental.availableSets += 1;
}
```
Найден и исправлен попутно, при рефакторинге `freeSpots` в вычисляемое значение (см. `docs/ba/data-model.md`, раздел «Важное архитектурное решение») — стало очевидно при пересмотре всей функции `cancelBooking`, что аналогичной защиты для проката не было.

## Промпты (сокращённо)
- Обнаружено самостоятельно при рефакторинге под добавление уведомлений (не было отдельного репорта от пользователя) — зафиксировано как «баг, найденный в этом проходе» прямо в комментарии кода

## Ручная проверка
- TC-24 (`docs/ba/test-cases.md`)
- Забронировал слот с прокатом (availableSets: 4→3), отменил бронь → availableSets вернулся к 4
