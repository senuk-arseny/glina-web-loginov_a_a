# Bug 04: Один номер телефона в разных форматах создавал двух разных клиентов

## Симптом
Найдено при анализе проекта перед написанием автотестов (не через ручной репорт пользователя). Номер `89001234567` и номер `+7 900 123-45-67` — один и тот же российский телефон, но `normalizePhone()` превращал их в разные ключи (`89001234567` и `79001234567`), из-за чего один и тот же человек получал два разных профиля в зависимости от того, как ввёл номер.

## Требование, которое нарушалось
User story 11 (`docs/ba/user-stories.md`) и заявленная в README гарантия «один телефон — один профиль».

## Причина
`normalizePhone()` просто убирала все нецифровые символы, не приводя ведущую цифру «8» к «7» для 11-значных российских номеров.

## Реализация фикса
```ts
function normalizePhone(phone: string): string {
  const digits = phone.replace(/\D/g, "");
  if (digits.length === 11 && digits.startsWith("8")) {
    return "7" + digits.slice(1);
  }
  return digits;
}
```

## Промпты (сокращённо)
- Обнаружено самостоятельно в рамках предварительного анализа перед написанием автотестов («изучи анализ проекта, выдели слабые места»)
- «Почини сразу вместе с тестами»

## Автоматическая проверка
`src/mock/api.test.ts` → `describe("loginOrRegister — регресс: разные форматы одного номера должны давать один профиль")`:
- номер с ведущей 8 и номер с +7 → один и тот же `clientId`
- два разных номера → два разных клиента

## Ручная проверка
Вошёл как «Арсений» / `89001234567`, вышел, вошёл заново как «Арсений» / `+7 900 123-45-67` → тот же профиль, тот же `visitedCount`.
