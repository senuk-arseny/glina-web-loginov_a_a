# Bug 05: Можно было встать в лист ожидания на занятие, уже отменённое мастерской

## Симптом
Найдено при анализе проекта перед написанием автотестов. `joinWaitlist()` проверял только количество свободных мест, но не статус слота — теоретически клиент мог встать в очередь на занятие со статусом `cancelled_by_studio`, хотя повторная запись на такой слот запрещена по R-008.

## Требование, которое нарушалось
FR-11 / R-008 (`docs/ba/functional-requirements.md`) — «повторная запись на отменённый слот запрещена». Лист ожидания — тоже форма записи, значит правило должно распространяться и на него.

## Причина
`joinWaitlist()` проверял `computeFreeSpots(slot) > 0`, но не проверял `slot.status`.

## Реализация фикса
```ts
export function joinWaitlist(clientId: string, slotId: string): WaitlistEntry {
  const slot = findSlot(slotId);
  if (slot.status === "cancelled_by_studio") {
    err("slot_cancelled", "Занятие отменено мастерской, запись в лист ожидания недоступна");
  }
  if (computeFreeSpots(slot) > 0) { ... }
  ...
}
```

## Промпты (сокращённо)
- Обнаружено самостоятельно в рамках предварительного анализа перед написанием автотестов
- «Почини сразу вместе с тестами»

## Автоматическая проверка
`src/mock/api.test.ts` → `describe("joinWaitlist — регресс: нельзя встать в очередь на отменённый слот")`:
- попытка встать в очередь на `slot-7` (отменён в сид-данных) → ошибка `slot_cancelled`
- позиция в очереди корректно считается по FIFO на обычном заполненном слоте

## Ручная проверка
Открыл слот со статусом «Отменено мастерской» — кнопка записи и так не показывается в UI (см. `SlotDetailsPage.tsx`), но прямой вызов API теперь тоже защищён на случай будущих изменений интерфейса.
