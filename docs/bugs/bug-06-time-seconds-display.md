# Bug 06: Секунды в отображении времени занятий

## Симптом
Время занятия отображалось с секундами: «Гончарный круг 05.07.2026, 18:55:50 · Павел Гончаров» — выглядит некорректно, пользователь явно указал на это как на визуальный баг.

## Требование, которое нарушалось
NFR-01 (`docs/ba/non-functional-requirements.md`) — интерфейс должен быть аккуратным и понятным; время с точностью до секунды не несёт пользы клиенту и выглядит как техническая недоработка.

## Причина
На двух страницах (`SlotDetailsPage.tsx`, `MyBookingsPage.tsx`) использовался голый `new Date(...).toLocaleString("ru-RU")` без явных опций форматирования — по умолчанию `toLocaleString` включает секунды. При этом на `SlotListPage.tsx` формат уже был задан явно (без секунд), поэтому расхождение было заметно только на части экранов.

## Реализация фикса
- Добавлена общая утилита `formatDateTime()` в `src/utils/format.ts` с явными опциями (`day`, `month`, `year`, `hour`, `minute` — без `second`)
- `SlotDetailsPage.tsx` и `MyBookingsPage.tsx` переведены на использование этой утилиты вместо голого `toLocaleString`

## Промпты (сокращённо)
- «ремя без секунд а только часы и минуты»

## Автоматическая проверка
`src/utils/format.test.ts` → `describe("formatDateTime — регресс bug (секунды в отображении времени)")`: проверяет, что отформатированная строка не содержит паттерна `:СС:СС`

## Ручная проверка
TC-25 (`docs/ba/test-cases.md`) — открыл «Мои записи» и детали занятия, время отображается как «05.07.2026, 21:55», без секунд.
