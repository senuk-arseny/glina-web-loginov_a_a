# Feature 02: Бронирование, прокат инструментов и лист ожидания

## Цель
Клиент бронирует место на занятие, опционально берёт инструменты напрокат; если мест нет — встаёт в лист ожидания и видит свою позицию в очереди.

## Требования
- FR-03, FR-04, FR-07, FR-08, FR-16 (`docs/ba/functional-requirements.md`)
- User story 3, 4, 6 (`docs/ba/user-stories.md`)
- UC-1, UC-3 (`docs/ba/use-cases.md`)

## Реализация
- `src/pages/SlotDetailsPage.tsx` — форма записи, чекбокс проката, кнопка «В лист ожидания» при freeSpots = 0
- `src/mock/api.ts` → `createBooking`, `joinWaitlist`, `getWaitlistPosition`, `listWaitlistForClient`, `hasActiveBooking`
- Лимит участников программы соблюдается за счёт того, что `freeSpots` — вычисляемое значение (`maxParticipants − активные брони`), а не хранимое поле
- Позиция в листе ожидания добавлена в `src/pages/MyBookingsPage.tsx` (изначально не была видна клиенту — закрытый пробел)

## Промпты (сокращённо)
- Промпт на бронирование с проверкой мест и проката
- Промпт на лист ожидания с продвижением очереди при отмене
- Отдельный запрос на добавление видимости позиции в очереди в UI (изначально клиент вставал в очередь "вслепую")
- Запрос на фикс дублирующей брони одним клиентом на один слот — см. `docs/bugs/bug-01-duplicate-booking.md`

## Ручная проверка
- TC-04–TC-09, TC-13–TC-16, TC-23 (`docs/ba/test-cases.md`) — пройдены
- Проверено вручную: повторная попытка записаться на уже забронированный слот блокируется и в API, и в UI (кнопка скрыта)
