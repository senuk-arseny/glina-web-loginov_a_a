# Feature 03: Отмена брони (ранняя/поздняя) и форс-мажор мастерской

## Цель
Клиент отменяет бронь; система различает раннюю (≥2ч) и позднюю (<2ч) отмену. Отдельно — реакция клиента на форс-мажорную отмену занятия мастерской (сломалась печь и т.п.).

## Требования
- FR-05, FR-06, FR-11 (`docs/ba/functional-requirements.md`)
- User story 5, 8 (`docs/ba/user-stories.md`)
- UC-2 (`docs/ba/use-cases.md`)
- R-008 (форс-мажор, из брифа)

## Реализация
- `src/pages/MyBookingsPage.tsx` — кнопка «Отменить», баннер предупреждения при поздней отмене
- `src/mock/api.ts` → `cancelBooking` — **вся логика определения ранняя/поздняя живёт на бэкенде**, клиент не вычисляет время сам (осознанное архитектурное решение, зафиксировано в `docs/ba/api-sequence.md`)
- `simulateStudioCancellation()` — демо-хелпер форс-мажора для проверки FR-11/FR-14 без реального бэкенда
- Восстановление фонда проката при отмене (изначально не реализовано, найдено при рефакторинге под уведомления) — см. `docs/bugs/bug-03-equipment-rental-leak.md`

## Промпты (сокращённо)
- Запрос на перенос вычисления ранней/поздней отмены с клиента на бэкенд (клиент изначально сам вычислял время — признано ненадёжным)
- Промпт на демо-хелпер форс-мажорной отмены с уведомлением затронутых клиентов
- Промпт на фикс утечки фонда проката при отмене

## Ручная проверка
- TC-10, TC-11, TC-12, TC-20, TC-24 (`docs/ba/test-cases.md`) — пройдены
- Проверено вручную: бронь с прокатом → отмена → фонд проката увеличился на 1
