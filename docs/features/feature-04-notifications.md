# Feature 04: Уведомления (подтверждение, напоминание, форс-мажор, лист ожидания)

## Цель
Клиент получает уведомления: подтверждение записи, напоминание перед занятием, отмена мастерской, освобождение места в листе ожидания. Изначально это требование (FR-12/13/14, отмечено Must) было пропущено при первой реализации — обнаружено при собственном ревью функциональных требований.

## Требования
- FR-12, FR-13, FR-14 (`docs/ba/functional-requirements.md`)
- User story 9 (`docs/ba/user-stories.md`)

## Реализация
- `src/pages/NotificationsPage.tsx` — список уведомлений, отметка «прочитано» при открытии
- `src/mock/api.ts` → `pushNotification`, `listNotifications`, `countUnreadNotifications`, `markAllNotificationsRead`, `generateDueReminders`
- Напоминания генерируются "по требованию" при открытии списка уведомлений (симуляция крон-джоба без реального фонового планировщика) — если до занятия осталось <24ч и напоминание ещё не отправлено
- Бейдж непрочитанных в навигации (`App.tsx`)

## Промпты (сокращённо)
- Запрос на реализацию всех трёх типов уведомлений как отдельного слоя данных
- Уточнение по способу генерации напоминаний без реального бэкенда/крона
- Промпт на бейдж непрочитанных в навигации

## Ручная проверка
- Проверено вручную: бронирование создаёт уведомление; отмена мастерской создаёт уведомление для всех активных броней на слот; лист ожидания уведомляет при продвижении очереди
- Открытие страницы уведомлений сбрасывает счётчик непрочитанных
