# Feature 05: Оценка мастера + вход по имени и телефону (без SMS)

## Цель
Две связанные, но независимые фичи, добавленные сверх обязательного минимума: (1) клиент оценивает мастера после занятия; (2) вход в приложение без пароля/SMS — по имени и телефону, что также стало точкой входа для лендинга.

## Требования
- FR-09, FR-10 (`docs/ba/functional-requirements.md`)
- User story 7, 11 (`docs/ba/user-stories.md`)
- UC-4, UC-5 (`docs/ba/use-cases.md`)

## Реализация
- `src/components/ReviewModal.tsx` + `MyBookingsPage.tsx` — форма оценки, баннер подтверждения после отправки (изначально отсутствовал — UX-баг, см. ниже)
- `src/mock/api.ts` → `createReview` — пересчёт среднего рейтинга мастера; округление до 2 знаков вместо 1 (иначе изменение рейтинга было незаметно при большом числе прошлых отзывов)
- `src/mock/api.ts` → `loginOrRegister`, `normalizePhone` — телефон как нормализованный ключ клиента
- `src/pages/RegisterPage.tsx`, `src/pages/HomePage.tsx` — лендинг и форма входа

## Промпты (сокращённо)
- Промпт на форму оценки с ограничением "одна оценка на бронь"
- Уточняющий запрос: "рейтинг не растёт" → объяснение архитектуры (расчёт на бэкенде) + фикс точности округления
- Промпт на добавление подтверждения после отправки отзыва (изначально молча закрывалась модалка)
- Промпт на дизайн-концепцию лендинга и формы входа, затем на реализацию

## Ручная проверка
- TC-17, TC-18, TC-19, TC-21, TC-22 (`docs/ba/test-cases.md`) — пройдены
- Проверено вручную: после отправки отзыва показывается баннер «Спасибо! Ваш отзыв отправлен...»
- Проверено вручную: вход тем же телефоном дважды возвращает один и тот же профиль
